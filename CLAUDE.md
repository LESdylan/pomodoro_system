# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

A Linux desktop Pomodoro timer written in C++. It runs as a background daemon that
cycles through work/break sessions, fires desktop notifications + sounds, and during
breaks spawns a separate fullscreen Qt overlay that grabs the keyboard/mouse to enforce
the break.

## Build & Run

```bash
make                 # build both binaries: pomodoro (daemon) + overlay_timer_qt (Qt overlay)
make pomodoro        # build ONLY the core daemon — no Qt toolchain needed
make re              # fclean + rebuild
make clean / fclean  # remove objects / objects + binaries

./pomodoro                                   # dev run, reads ./config/settings.cfg (must run from repo root)
./pomodoro --config "config/work_duration=1.conf"   # quick test: 1-minute work session
./pomodoro --daemon --config <path>          # daemon mode (logs to ~/.local/share/pomodoro-timer/logs/)
./overlay_timer_qt 10 "Skip? (Y/N)"          # run the break overlay standalone for 10s (testing)
```

There is **no test suite** and no linter configured.

### Toolchain constraints
- The core daemon (`pomodoro`) is compiled with **`-std=c++98`** (see `Makefile`). Do not use
  C++11+ features in `main.cpp`, `timer.*`, `notification.*`, `config.*` — this is why the code
  has hand-rolled `intToString`/`stringToInt` helpers instead of `std::to_string`/`std::stoi`.
- `overlay_timer_qt` is built via **qmake (Qt5 / QtWidgets)**, defined by `overlay_timer_qt.pro`.
  It uses `QDesktopWidget`, which was **removed in Qt6** — building requires Qt5. The Makefile
  invokes `qmake` then a nested `make -f Makefile.qt` and renames the output.
- Runtime depends on external commands shelled out via `system()`: `notify-send` (libnotify),
  `paplay` (PulseAudio) / `aplay` (ALSA), and optionally `speaker-test`.
- Sound assets live in the **`collection_sounds/` git submodule** — run
  `git submodule update --init` after cloning.

## systemd user service

```bash
make install-service   # or ./install-service.sh — copies binaries/config/sounds, generates & starts the service
make uninstall-service
systemctl --user status|restart|stop pomodoro-timer.service
journalctl --user -u pomodoro-timer.service -f
```

`install-service.sh` deploys to `~/.local/share/pomodoro-timer/{bin,logs,sounds}` and config to
`~/.config/pomodoro-timer/settings.cfg`. The root-level `pomodoro-timer.service` file is a stub;
the real unit is generated by the install script. `fix-daemon-audio.sh`, `update-daemon-config.sh`,
and `uninstall-daemon.sh` are convenience wrappers for re-deploying config / fixing PulseAudio env.

## Architecture

**Two-process design.** The daemon and the break overlay are separate binaries that communicate
only by process lifecycle (fork + wait), not IPC.

- `src/main.cpp` — entry point + the session state machine. `runDaemon()` is the core loop:
  work session → notify/sound → `Timer` countdown → on every Nth cycle pick long vs short break →
  notify/sound → `runOverlay()` → break countdown → "back to work". Also handles daemon-mode
  detection (`--daemon`), config-path parsing (`--config`), signal handling (SIGTERM/INT/QUIT),
  and file+console logging.
- `src/config.{hpp,cpp}` — `Config` class: the single source of all durations, messages, sounds,
  and the overlay prompt. Parses a flat `key=value` file.
- `src/timer.{h,cpp}` — `Timer` class: **polling busy-wait**. `waitForCompletion()` loops
  `sleep(1)` comparing `time(0)` against a start timestamp; there is no event/threading.
- `src/notification.{h,cpp}` — `Notification` class: shells out to `notify-send` and tries a
  fallback chain of `paplay` → `aplay` → hardcoded system sounds → terminal bell.
- `src/overlay_timer_qt.cpp` — **standalone Qt app** (its own `main()`). `OverlayManager` creates one
  `OverlayTimer` widget per screen, frameless + always-on-top, and `grabKeyboard()`/`grabMouse()` to
  block input. ESC opens a confirm prompt; Y skips the break. Heavily custom `QPainter` rendering.
- `src/overlay_timer.cpp` — **legacy/unused**. A raw Xlib/Xinerama implementation of the overlay,
  NOT compiled by the Makefile. The Qt version superseded it. Ignore unless explicitly reviving X11.

### How breaks are enforced (cross-file behavior worth knowing)

In `runDaemon()`, a break does two sequential full-length waits:
1. `runOverlay(breakDuration * 60, ...)` — forks `overlay_timer_qt` and **blocks on `waitpid`** while
   the overlay counts down the full break.
2. Then `timer.startBreak()` + `timer.waitForCompletion()` — waits the **full break duration again**.

So wall-clock break time is effectively ~2× the configured duration, and skipping the overlay early
(ESC→Y) still leaves the second `Timer` wait. Keep this in mind before changing break timing.

## Configuration system

Config files are flat `key=value` (see `config/settings.cfg`). Parsing in `Config::loadSettings()`
uses prefix matching with **hardcoded `substr()` offsets** equal to each key's length, e.g.
`line.find("work_duration=") == 0` → `line.substr(14)`. **Adding or renaming a key requires editing
four places consistently:** the member + default in the `Config` constructor (`config.cpp`), a parse
branch with the correct substr offset in `loadSettings()`, a write line in `saveSettings()`, and a
getter. Off-by-one substr offsets are the most likely bug.

Recognized keys: `work_duration`, `break_duration`, `long_break_duration`,
`sessions_before_long_break`, `notification_enabled`, `notification_sound`, `work_message`,
`break_message`, `long_break_message`, `work_sound`, `break_sound`, `long_break_sound`,
`overlay_prompt`. Durations are in minutes.

### Sound resolution caveat

`Config::constructSoundPath()` searches `~/.local/share/pomodoro-timer/sounds/`, a hardcoded
(and stale) `~/Documents/system/pomodoro_system/sounds/`, then `./sounds/`, then cwd. **No `sounds/`
directory exists in the repo** — the actual `.wav` files are in the `collection_sounds/` submodule,
which none of those paths point to, and `install-service.sh` copies from a non-existent `./sounds/`.
In practice playback usually falls through to the system-sound fallback chain in
`notification.cpp`. If wiring real sounds, reconcile these paths (e.g. symlink or copy
`collection_sounds/` → `sounds/`).
